
Gastenboek Game Developer
Profiel: Game Developer
Focus: Gamification met XP, Levels en Upvotes
Beschreven volgens de MoSCoW methode:

Must haves
Basis functionaliteit
De standaard pagina van het gastenboek is een overzicht van alle berichten
Het gastenboek is een openbare pagina, om berichten te zien is inloggen niet nodig
Om in te kunnen loggen moet je een account aan kunnen maken met username / password
Om een bericht te plaatsen moet je inloggen, pas dan kan een bericht worden geschreven
Per sessie mag een gebruiker maar één keer een bericht achterlaten
Een bericht van een bezoeker moet bestaan uit:
De naam van de gebruiker
Datum en tijd van plaatsing (formaat: dd-mm-yyyy hh:mm) - deze moet van de server komen
Het bericht zelf
Een afbeelding bij het bericht
De styling moet aantrekkelijk zijn en werken op desktop en mobiel (responsive)
De website moet een default landingspage bevatten
Data wordt opgeslagen in een MySQL database
Game Developer specifiek Nieuw
XP & Levels systeem: Gebruikers hebben XP (experience points) en een level dat stijgt naarmate ze meer XP verzamelen
XP verdienen: Acties geven XP, bijvoorbeeld:
Een bericht plaatsen
Upvotes ontvangen op je berichten
Upvote systeem: Ingelogde gebruikers kunnen berichten van anderen upvoten (liken)
Should haves
CRUD functionaliteit:
Een bericht dat is geplaatst moet kunnen worden gewijzigd of gewist
Denk hierbij aan bevestigingsdialogs ("Weet je het zeker?")
Alleen eigen berichten kunnen worden gewijzigd of gewist
Leaderboard: Een overzichtspagina met de top gebruikers gesorteerd op XP/level
Level-badges: Bij berichten wordt het level en/of een badge van de gebruiker getoond
Could haves
Achievements/badges systeem: Speciale badges voor mijlpalen, bijvoorbeeld:
"Eerste bericht"
"10 upvotes ontvangen"
"Level 5 bereikt"
Dagelijkse login streak: Bonus XP voor opeenvolgende dagen inloggen
Coins/virtuele valuta: Een tweede valuta naast XP waarmee gebruikers iets kunnen "kopen" (bijv. profielflair)
Won't haves
Reacties kunnen plaatsen op berichten
De mogelijkheid om berichten te delen op sociale media (Facebook, Twitter, etc.)
Optie om in te loggen met sociale media accounts (third party login)

Beoordeling
De volgende activiteiten binnen deze taak worden beoordeeld:

Het design, vormgeving en gebruikersvriendelijkheid van je gastenboek
De technische werking van je gastenboek
De wijze waarop je het gastenboek presenteert en reflecteert
User Stories
User story 01: Als ontwikkelaar wil ik het programma van eisen doornemen, zodat ik meer kennis over de opdracht verkrijg.
User story 02: Als ontwikkelaar wil ik vragen kunnen stellen aan de docent/klant (klantgesprek), zodat ik meer kennis over de opdracht krijg.
User story 03: Als ontwikkelaar wil ik een basisontwerp van de pagina maken, zodat ik een start met de homepage kan maken.
User story 04: Als ontwikkelaar wil ik mijn IDE inrichten inclusief Git(hub) Classroom repository, zodat ik gestructureerd aan het project kan werken.
User story 05: Als ontwikkelaar wil ik de diverse assets (afbeeldingen/logos/lettertypes) verzamelen, zodat ik deze kan gebruiken in het project.
User story 06: Als bezoeker wil ik een aantrekkelijke frontpage (default landingspage) zien, zodat ik een goede eerste indruk krijg van het gastenboek.
User story 07: Als ontwikkelaar wil ik een datamodel/database ontwerpen voor berichten, gebruikers en XP/levels, zodat de data gestructureerd kan worden opgeslagen.
User story 08: Als ontwikkelaar wil ik backend functies maken voor opslag en weergave van berichten, zodat de applicatie correct functioneert.
User story 09: Als ontwikkelaar wil ik de backend functies koppelen aan de frontend pagina, zodat de data op de pagina getoond kan worden.
User story 10: Als gebruiker wil ik een veilig inlogsysteem gebruiken, zodat mijn account beschermd is.
User story 11: Als ingelogde gebruiker wil ik per sessie slechts één bericht kunnen plaatsen, zodat spam wordt voorkomen.
User story 12: Als bezoeker wil ik de website goed kunnen gebruiken op zowel desktop als mobiel (responsive), zodat ik altijd een goede gebruikerservaring heb.
User story 13: Als gebruiker wil ik mijn eigen berichten kunnen aanmaken, lezen, wijzigen en verwijderen (CRUD), zodat ik volledige controle heb over mijn content.
User story 14: [Game Dev] Als gebruiker wil ik XP (experience points) verdienen en levels behalen, zodat mijn activiteit en progressie zichtbaar worden beloond.
User story 15: [Game Dev] Als ingelogde gebruiker wil ik berichten van anderen kunnen upvoten, zodat ik mijn waardering kan tonen en de auteur XP kan laten verdienen.
User story 16: [Game Dev] Als bezoeker wil ik een leaderboard pagina kunnen bekijken, zodat ik kan zien wie de meest actieve gebruikers zijn gesorteerd op XP/level.
User story 17: Als ontwikkelaar wil ik de applicatie testen, zodat eventuele fouten kunnen worden afgevangen en opgelost.
User story 18: Als ontwikkelaar wil ik de applicatie opleveren op de VPS, zodat het gastenboek openbaar gebruikt kan worden.
